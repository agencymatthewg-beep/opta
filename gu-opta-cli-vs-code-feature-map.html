Architectural Principle
The CLI daemon owns all intelligence — session orchestration, agent loops, tool execution, browser automation, permission gating, and LMX routing. Code Desktop is an intentionally thin UI layer that connects to the daemon over HTTP REST + WebSocket and delegates every AI operation back through it. This means Desktop features are only as capable as the daemon version they're connected to.

Key Insights
CLI is the Intelligence Layer
All agentic capabilities — browser automation, web search, embeddings, git operations, LSP, learning — live exclusively in the CLI daemon. Code Desktop is a presentation surface. To unlock any capability in Desktop, it must first exist in the daemon operation registry.
Desktop is the Experience Layer
Desktop owns the visual OS: spring-animated studios, glass panel hierarchy, the OPTA text logo system, fullscreen transitions, keyboard chord navigation, and real-time dashboards. None of these exist in CLI — they're native Tauri + Framer Motion capabilities.
Shared via Daemon Protocol
Features that appear in both surfaces (model management, MCP, sessions, vault, permissions) go through the daemon's v3 HTTP REST + WebSocket protocol. Desktop calls daemonClient.runOperation() for mutations and subscribes to WS for streaming events.
Mono512 as the Inference Node
Both CLI and Desktop route inference to Opta LMX at 192.168.188.11:1234 (Mono512 Mac Studio, 512GB RAM). MacBook (Opta48) is client-only — never run models locally. The Desktop Models Studio VRAM bar reflects Mono512 unified memory, not the MacBook.
