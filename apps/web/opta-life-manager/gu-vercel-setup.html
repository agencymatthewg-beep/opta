Vercel + GitHub Setup

Complete guide for deploying Opta Life Manager with automatic CI/CD

+

Why This Setup?

Connecting GitHub to Vercel gives you the best developer experience for scaling:

Auto Deployments

Every git push automatically deploys. No CLI commands needed.

Preview URLs

Every branch/PR gets its own preview URL for testing.

Instant Rollbacks

One-click rollback to any previous deployment.

No Rate Limits

Git-based deploys bypass CLI upload limits.

1

Connect GitHub Account

On the Vercel dashboard (vercel.com/new):

  • Click Continue with GitHub
  • Authorize Vercel to access your GitHub account
  • Select All repositories or just agencymatthewg-beep/opta
2

Import Repository

Find and select your repository:

  • Search for opta in the repository list
  • Click Import next to agencymatthewg-beep/opta
3

Configure Project Settings

Set these critical configuration options:

Important: You must set the Root Directory since the app is in a subfolder.
Setting Value
Project Name opta-life-manager
Framework Preset Next.js (auto-detected)
Root Directory opta-life-manager
Build Command npm run build (default)
Output Directory Leave empty (Next.js default)
4

Add Environment Variables

Expand "Environment Variables" and add these (get values from your .env.local):

Variable Required Description
AUTH_SECRET Yes NextAuth secret key
AUTH_GOOGLE_ID Yes Google OAuth Client ID
AUTH_GOOGLE_SECRET Yes Google OAuth Client Secret
GEMINI_API_KEY For AI Google Gemini API key
TODOIST_CLIENT_ID Optional Todoist integration
TODOIST_CLIENT_SECRET Optional Todoist integration
# Generate a new AUTH_SECRET: openssl rand -base64 32
5

Deploy

  • Click Deploy
  • Wait 2-3 minutes for the build to complete
  • You'll get a URL like opta-life-manager-xxx.vercel.app
Success! Your app is now live with automatic deployments enabled.
6

Add Custom Domain

Connect lm.optamize.biz to your deployment:

  • Go to your project dashboard on Vercel
  • Click Settings > Domains
  • Enter lm.optamize.biz and click Add
  • Vercel will show DNS configuration needed
DNS Setup: You'll need to add a CNAME record in Cloudflare:
Type: CNAME Name: lm Target: cname.vercel-dns.com Proxy: DNS only (grey cloud)
7

Update Google OAuth

Add your new domain to Google Cloud Console:

  • Go to Google Cloud Console > Credentials
  • Edit your OAuth 2.0 Client ID
  • Add to Authorized JavaScript origins:
https://lm.optamize.biz
  • Add to Authorized redirect URIs:
https://lm.optamize.biz/api/auth/callback/google
*

Future Expansion Ready

With this setup, you can easily add more apps:

Subdomain Purpose Root Directory
lm.optamize.biz Life Manager (this app) opta-life-manager
app.optamize.biz Future: Main web app opta-web
api.optamize.biz Future: API service opta-api
docs.optamize.biz Future: Documentation opta-docs

Each can be a separate Vercel project from the same monorepo, or separate repos as you scale.
