03
Path Split-Brain — Bug Fixed
🐛 Before (split-brain)
Two independent opta_config_dir() implementations
daemon_ops.rs used runtime env-var chain:
XDG_CONFIG_HOME → APPDATA → HOME/.config

setup_wizard.rs used compile-time #[cfg(target_os)] branching.

On Windows with XDG_CONFIG_HOME set, the two modules resolved to different directories — daemon would write config to XDG path, setup wizard would read from APPDATA. Silent data loss.
→
✅ After (config_paths.rs)
Single module, explicit compile-time branching
#[cfg(target_os = "macos")] { $HOME/.config/opta } #[cfg(target_os = "windows")] { %APPDATA%\opta } #[cfg(not(any(...)))] // Linux { $XDG_CONFIG_HOME/opta || $HOME/.config/opta }
Both daemon_ops and setup_wizard call crate::config_paths::opta_config_dir(). Can never diverge. Includes a test: opta_config_dir_returns_ok().

🔒 Cross-Platform Release Gate
opta-code-release-cross-platform-gate.yml triggers on opta-code-v* tags. It polls until BOTH macOS DMG and Windows MSI artifacts are uploaded, then validates they're distinct HTTPS URLs. Only then does it update channels/beta.json or channels/stable.json in Opta Init. A release cannot ship on just one platform.
07
Code Signing & Release Flow
🍎 macOS Signing (Apple Developer ID)
Secret: APPLE_CERTIFICATE (base64 .p12)
Secret: APPLE_CERTIFICATE_PASSWORD
Secret: APPLE_SIGNING_IDENTITY
Secret: APPLE_ID, APPLE_PASSWORD, APPLE_TEAM_ID
⚠️ When secrets not set → builds unsigned (OK for dev). Release tag requires all secrets or CI fails.
🪟 Windows Signing (Authenticode)
Secret: WINDOWS_CERTIFICATE (base64 PFX)
Secret: WINDOWS_CERTIFICATE_PASSWORD
PFX → imported to runner Keychain → thumbprint patched into tauri.conf.json
🔴 Blocker: No Authenticode certificate yet. Windows builds are unsigned. SmartScreen warnings will appear for users until an EV cert is obtained.
