Design Mission

No tiles. No boxes. Seamless embedding.

Opta Help should feel like reading text laid directly on the void canvas. Surfaces are atmospheric, not boxed. Separators are gone or replaced with gradient fades. Interactive regions use glass depth, not borders, to establish hierarchy. The result is a premium dark-ops aesthetic — content-first, chrome-light.

Background: atmospheric gradients + starfield Surfaces: .doc-embed-block pseudo glow Borders: transparent or gradient-fade only Typography: .prose-opta, dark only

Constraints

Hard Rules for This Work

No unrelated changes

This is a large multi-project workspace. Only touch 1U-Opta-Help files.

Never touch OptaRing

shared/OptaRing.tsx CSS keyframes are sacred. Do not modify.

Static export

No API routes, no server actions. output: 'export' in next.config.ts.

CSS variables only

Never write hex/rgb literals in component code. Use var(--color-*).

npm (not pnpm)

This app uses npm. Do not use pnpm or bun inside 1U-Opta-Help.

Readability first

Remove chrome, not contrast. Preserve spacing, hierarchy, and text legibility.
