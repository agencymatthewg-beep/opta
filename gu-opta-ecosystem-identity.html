/ fixes applied · 2026-03-06

Corrections Made This Session

🔧 CLI Atpo colour corrected: SettingsOverlay.tsx PAGES array had atpo: '#c084fc' (light purple). Corrected to #f472b6 (canonical pink). The old colour conflicted with the Opta brand palette and didn't match the Atpo Studio identity established in the Desktop.
🔧 CLI Advanced colour freed: advanced: '#22d3ee' was using the Browser identity cyan for an unrelated "Advanced" settings category. Changed to #94a3b8 (slate) to free cyan exclusively for Browser-related UI.
✓ Feature identity tokens added to palette.ts: TUI_COLORS.browser, TUI_COLORS.models, TUI_COLORS.atpo are now the canonical source for feature colours in all CLI TUI components. Import from palette.ts — don't hardcode hex in TUI components.

What was changed

// palette.ts — BEFORE
export const TUI_COLORS = {
  info: '#22d3ee', // cyan — general info colour, no feature identity
  ...
// (no browser / models / atpo entries)
}

// palette.ts — AFTER ✓
export const TUI_COLORS = {
  info: '#22d3ee', // still exists as general semantic colour
  ...
  // Feature identity colours — canonical, synced with desktop + web
  browser: '#22d3ee', // Opta Browser
  models: '#a78bfa', // Opta Models
  atpo: '#f472b6', // Opta Atpo
}
// SettingsOverlay.tsx PAGES — BEFORE (wrong atpo + misused cyan)
{ id: 'advanced', label: 'Advanced', color: '#22d3ee' }, // ← browser cyan on wrong thing
{ id: 'atpo', label: 'Atpo', color: '#c084fc' }, // ← wrong colour for Atpo

// SettingsOverlay.tsx PAGES — AFTER ✓
{ id: 'advanced', label: 'Advanced', color: '#94a3b8' }, // slate — neutral, frees cyan
{ id: 'atpo', label: 'Atpo', color: '#f472b6' }, // ← canonical Atpo pink ✓
/ usage · token reference

How to Use These Tokens

In Web Apps (CSS)

/* @import once in your app's global CSS */
@import 'design/studio-system/identity/ecosystem-identity-tokens.css';

/* Then use in any component: */
.browser-card { border-color: var(--opta-feature-browser); }
.models-badge { background: var(--opta-feature-models-soft); color: var(--opta-feature-models-text); }
.atpo-section { border-left: 3px solid var(--opta-feature-atpo); }

In CLI TUI Components (TypeScript/Ink)

import { TUI_COLORS } from './palette.js';
import chalk from 'chalk';

// Use feature identity colours directly
const browserLabel = chalk.hex(TUI_COLORS.browser)('Browser');
const modelsLabel = chalk.hex(TUI_COLORS.models)('Models');
const atpoLabel = chalk.hex(TUI_COLORS.atpo)('Atpo');

// In Ink/React JSX:
<Text color={TUI_COLORS.browser}>Browser</Text>
<Text color={TUI_COLORS.models}>Models</Text>
<Text color={TUI_COLORS.atpo}>Atpo</Text>

In Opta Learn Guide Cards

// Tag guides with data-feature for automatic colour routing
<div className="guide-card" data-feature="browser"> ← cyan accent</div>
<div className="guide-card" data-feature="models"> ← violet accent</div>
<div className="guide-card" data-feature="atpo"> ← pink accent</div>

Full Colour Hierarchy

The complete picture of how Opta colours layer from brand foundation to feature identity.

/* ── Level 1: Opta Brand Foundation ── */
--opta-primary: #8b5cf6; /* Electric Violet — brand identity, CTAs */
--opta-primary-glow: #a855f7; /* Settings Studio, logo glow */

/* ── Level 2: Studio Atmosphere ── */
--studio-atmo-0: #040b18; /* deep space blue — any Studio backdrop */
--studio-border: rgba(167,204,255,0.24); /* cool blue glass borders */

/* ── Level 3: Feature Identity (cross-ecosystem) ── */
--opta-feature-browser: #22d3ee; /* Browser — all surfaces */
--opta-feature-models: #a78bfa; /* Models — all surfaces */
--opta-feature-atpo: #f472b6; /* Atpo — all surfaces */

/* ── Level 4: State / Semantic (universal) ── */
--opta-success: #10b981; /* emerald — success states */
--opta-warning: #f59e0b; /* amber — warnings */
--opta-danger: #ef4444; /* rose — errors */
