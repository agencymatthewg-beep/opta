Source Attribution System Ideas

Inline Source Badges

Display small clickable badges next to each data point (e.g., [HF] for Hugging Face, [LMS] for LMSys). Clicking opens source in new tab with exact URL.

UI Critical

"Last Updated" Timestamps

Show relative timestamps ("2h ago", "Yesterday") for each data field. Users can see data freshness at a glance. Tooltip shows exact UTC datetime.

UI Critical

Source Confidence Indicators

Rate source reliability (Official API = High, Scraped = Medium, User Submitted = Low). Display as color-coded dots or shield icons.

UI Enhancement

Data Provenance Modal

Click any data point to see full provenance: original source URL, fetch timestamp, transformation applied, historical values. Full audit trail.

Modal Enhancement

Methodology Explainers

For each benchmark, provide expandable "How is this measured?" sections explaining the methodology, dataset used, and known limitations.

Educational

Community Corrections

Allow users to flag potentially incorrect data and submit corrections with source URLs. Moderated queue for verification before updates.

Community Future

Key Takeaways

  • Start with Hugging Face - Best API, most comprehensive benchmark data, high trust
  • Source attribution is non-negotiable - Every data point needs a clickable source and timestamp
  • Search & Compare are core features - These differentiate AI Comp from static leaderboards
  • Normalize everything - Different sources use different units, schemas; unified data layer is critical
  • Keep history - Store all historical data for trend analysis and audit trails
  • Progressive enhancement - Start with API sources, add scraping later for broader coverage
