⚠ SettingsModal.tsx at 2251 Lines
A single component managing 30+ settings categories, parallel daemon fetches, LMX discovery, connection probing, 12 distinct tab renders, and form state is doing too much. It currently works, but will become a maintenance burden. Recommend extracting settings state into a custom hook and splitting tab components into their own module.

⚠ Massive Promise.all in loadRemoteSettings
32 parallel daemonClient.configGet() calls with individual .catch() fallbacks fire every time the settings modal opens. No debounce, no timeout, no stale-while-revalidate. If the daemon is slow, the entire settings page is blocked. A single configGetBulk endpoint would reduce this to one HTTP round-trip.

⚠ Scroll Reset Triple-Timeout Pattern
Tab switch scroll reset uses RAF + setTimeout(80ms) + setTimeout(180ms) — three sequential fallbacks. This is a code smell that likely patches a layout timing race. The correct fix is a ResizeObserver or a useLayoutEffect that waits for the content panel to settle before scrolling to top.

The Multi-Studio System

The Settings Studio has established a complete, polished UI pattern — glass shell, fullscreen toggle, Opta Text Logo animation, 3-layer navigation, and a unified keyboard system. The idea is to extract this into a StudioOverlay base and build purpose-built Studios that share 100% of the chrome while providing optimal UX for their specific domain.

💡 Core Principle: Every Studio shares the same glass aesthetic, fullscreen capability, Opta Text Logo animation (with its own unique sub-label in Press Start 2P), layer transitions, and keyboard bindings. Only the content, accent colour, and category structure differ. Users build muscle memory once; it works everywhere.

✓ Zero new animation code needed. The existing .opta-studio-logo-reserve with max-height transition + .opta-studio-logo-settings class is the system. For new Studios, simply add:
.opta-studio-logo-browser, .opta-studio-logo-models, .opta-studio-logo-atpo
with the studio's accent colour in place of --opta-primary-glow. The animation is fully CSS — no JS changes.
