What the token file will contain

/* motion/logo-animation.css — Opta Text Logo complete spec */

:root {
/* Font */
--studio-logo-font: 'Press Start 2P', monospace;
--studio-logo-size: 2.2rem;
--studio-logo-sub-size: 1.1rem;

/* Shadow recipe — uses [accent] slot */
--studio-logo-shadow-base: 2px 2px 0 #09090b, 3px 3px 0 #09090b;
--studio-logo-shadow-accent: 4px 4px 0 var(--studio-accent), 5px 5px 0 var(--studio-accent);
--studio-logo-shadow-cap: 7px 7px 0 #09090b, 8px 8px 0 #09090b;

/* Stagger timing */
--studio-logo-stagger: 26ms; /* per letter delay increment */

/* Animation duration (matches opta-transition-logo-duration) */
--studio-logo-duration: var(--opta-transition-logo-duration, 340ms);
--studio-logo-ease: var(--opta-transition-logo-ease, cubic-bezier(0.22, 1, 0.36, 1));

/* Sub-label reserve (max-height transition) */
--studio-logo-reserve-collapsed: 0px;
--studio-logo-reserve-expanded: 2.5rem;
}
/ plan · workflow with gemini

The Gemini Design Workflow

Two context files let you and Gemini collaborate on new Studio designs without re-explaining the system each time. Gemini gets the full spec; you give it one creative direction.

1
Create the Design Library Do this once
Build design/studio-system/ with the extracted token files, component HTML demos, and Gemini context markdowns. This is the investment — done once, reused forever.
2
Open Gemini — Paste Context File
Drop GEMINI-CONTEXT-STUDIO.md into the Gemini prompt. Gemini now knows: the full token set, the glass shell spec, all transition values, the keyboard system, and the logo recipe. Zero re-explanation needed.
3
Give Gemini One Design Brief
e.g. "Design the L2 grid for the Browser Studio — 7 categories, cyan accent, optimal layout for managing localhost endpoints and browser sessions." Gemini generates within the established system constraints.
4
Gemini Outputs HTML Demo → Review in Browser
Gemini generates a self-contained HTML mockup. Open it in browser — it inherits all the Studio tokens automatically because the context includes the :root CSS block.
5
You + Claude: Implement into the Codebase
Approved design goes from Gemini HTML → code. Claude implements using the studio-template/ files as the starting point, replacing only content — not re-engineering the chrome.
💡 Key insight: Gemini excels at design generation when given precise system constraints. The more specific the context file (exact pixel values, exact token names, exact easing curves), the more on-system the output. The GEMINI-CONTEXT-STUDIO.md is dense by design — it tells Gemini exactly what NOT to change (glass formula, logo system, transition timings) and what it can freely create (category content, iconography, layout density).
