Core Innovations

Context Distillation

Each executor receives a focused 30-50k token "context packet" instead of full conversation history. The oap-distiller agent extracts only what's needed for each task.

OAP Original - Addresses context window bloat in long sessions

Wave-Based Parallel Execution

Tasks are grouped by dependency waves - all tasks in a wave execute simultaneously. Computed from YAML plan dependencies using topological sorting.

Enhanced from GSD - Added dependency computation and wave visualization

Plan-as-Code (YAML)

Machine-readable YAML plans with explicit contracts, verification commands, and computed dependencies. Enables validation and tooling integration.

OAP Original - Replaces prose-based PLAN.md files

Mass Question Refinement

15-25 targeted questions eliminate ambiguity before execution. The oap-interrogator ensures zero guesswork during implementation.

OAP Original - Systematic pre-planning clarification

Specialized Agent Frameworks

9 agents with distinct cognitive frameworks: EXPAND, CLARIFY, STRUCTURE, EXTRACT, DELIVER, MERGE, DISCOVER, VERIFY, INVESTIGATE.

OAP Original - Phase-optimized agent specialization

Hypothesis-Driven Debugging

Systematic debugging with persistent DEBUG.yaml state that survives context resets. Generate, test, and track hypotheses methodically.

Inspired by /gsd:debug - Added YAML persistence and hypothesis tracking

Feature Inspirations & Innovations

OAP was built by analyzing GSD's 25 commands and identifying 18 missing features. The 8 highest-priority features were redesigned with OAP's innovations.

Inspired by /gsd:new-project

/oap:new-project

Deep project initialization with context gathering

OAP Innovation
Uses oap-interrogator for systematic questioning. Outputs config.yaml (YAML) instead of JSON. PROJECT.md with structured sections.
Inspired by /gsd:create-roadmap

/oap:roadmap

Define all phases with domain expertise detection

OAP Innovation
Uses oap-visionary for creative exploration. YAML roadmap format (ROADMAP.yaml) with machine-readable phase definitions.
Inspired by /gsd:research-phase

/oap:research

Research complex/niche domains before planning

OAP Innovation
New oap-researcher agent with DISCOVER framework. Specialized for security, payments, real-time, and other complex domains.
Inspired by /gsd:map-codebase

/oap:map

Parallel codebase analysis for brownfield projects

OAP Innovation
Uses parallel oap-distiller variants. Produces structured ARCHITECTURE.md, STACK.md, TESTING.md, ISSUES.md.
Inspired by /gsd:verify-work

/oap:verify

Guide UAT testing and capture issues

OAP Innovation
New oap-tester agent with VERIFY framework. Structured ISSUES.yaml with severity levels and reproduction steps.
Inspired by /gsd:plan-fix

/oap:fix

Create fix plans from UAT issues

OAP Innovation
Uses oap-compiler to generate YAML fix plans. Links to ISSUES.yaml for traceability. Same format as regular plans.
Inspired by /gsd:debug

/oap:debug

Systematic debugging with persistent state

OAP Innovation
New oap-debugger agent with INVESTIGATE framework. Persistent DEBUG.yaml tracks hypotheses, tests, and evidence.
Inspired by /gsd:resume-project

/oap:resume

Session continuity and context restoration

OAP Innovation
Uses oap-synthesizer to reconstruct context. Analyzes STATE.md, ROADMAP.yaml, and phase files. Routes to next action.

Potential Improvements

Based on recent Claude Code updates and best practices, these enhancements could further improve OAP:

1. MCP Server Integration

Expose OAP commands as an MCP server for IDE integration. Would allow VSCode/Cursor to invoke /oap:execute directly.

Source: Claude Code MCP architecture

2. GenUI Progress Dashboard

Replace text-based /oap:status with rich GenUI HTML dashboard showing wave progress, agent status, and real-time execution.

Source: GSD's /gsd:progress GenUI implementation

3. Plan Validation Schema

Add JSON Schema validation for all YAML files. Could use Zod or Ajv to validate plans before execution.

Source: YAML schema validation best practices

4. Milestone Management

Add /oap:milestone for versioned milestones (v1.0, v2.0). Includes archive, git tagging, and PROJECT.md evolution.

Source: GSD's /gsd:new-milestone and /gsd:complete-milestone

5. Decimal Phase Insertion

Add /oap:insert-phase for urgent work insertion as decimal phases (e.g., 72.1 between 72 and 73).

Source: GSD's /gsd:insert-phase

6. Context Packet Caching

Cache distilled context packets to avoid regeneration. Hash-based invalidation when source files change.

Source: Performance optimization for parallel execution

7. Agent Telemetry

Track agent performance metrics (token usage, execution time, success rate) for optimization insights.

Source: Observability best practices

8. Interactive Plan Editor

Web-based YAML plan editor with drag-drop task reordering, dependency visualization, and wave preview.

Source: Developer experience enhancement
