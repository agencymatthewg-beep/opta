Unique Strengths

Competitive Edge

Native Local LLM

Purpose-built for Apple Silicon + MLX inference via Opta LMX. Zero cloud dependency. Full privacy. No API costs. Only CLI with native local-first architecture.

5-Mode Permissions

Most granular permission system: safe, auto, plan (read-only), dangerous (no gates), ci (hard deny). Claude Code has 3. Most others: binary Y/N.

3-Level Circuit Breaker

Progressive safety: warn@20, pause@40, hard-stop@100 tool calls. Configurable per-level. Only agent with progressive runaway protection.

Observation Masking

JetBrains NeurIPS 2025 technique. Replaces old tool results with compact placeholders. 52% context savings with zero API overhead. No other CLI implements this.

14 Tools + MCP

Second-highest built-in tool count (Claude Code: 16). Plus full MCP extensibility for unlimited external tools. Ahead of Gemini CLI (12), Cursor (~10), all others.

Plan Mode

Read-only exploration mode with tool filtering. Only Opta and Claude Code offer this. System prompt injection guides the model toward analysis-only behavior.

⚠

Remaining Gaps

vs. Leader

No Sub-Agents

Claude Code's Task tool spawns parallel sub-agents for complex work. Opta runs single-threaded. This is the biggest capability gap for ambitious tasks.

No Background Shell

Claude Code can run long processes in background while continuing work. Opta blocks on shell execution. Limits parallel workflows.

No Notebook Support

Claude Code and Cursor can edit Jupyter .ipynb cells directly. Opta treats notebooks as opaque files. Important for data science workflows.

No LSP Integration

Claude Code uses LSP for go-to-definition, find-references, hover. Opta relies on regex search. LSP would give precise code intelligence.

No Hook System

Claude Code supports user-defined hooks (pre/post tool execution). Enables custom validation, logging, and workflow automation. Opta has no hook mechanism.

No Repo Map

Aider generates structural repo maps for context. Opta's export map is simpler. A full repo map would improve code navigation for large codebases.

V3 Roadmap

5 Phases

Prioritized by competitive impact. Each phase independently shippable. Target: close the Agent Depth gap (45 → 80) and reach composite parity with Claude Code.

1 Background Shell + Async Execution

  • Non-blocking shell execution with output polling
  • Process management (list, kill, get output)
  • Long-running task support (builds, tests, servers)
  • Timeout configuration per command

2 LSP Integration

  • Language Server Protocol client for TypeScript, Python, Go
  • go-to-definition, find-references, hover tools
  • Workspace symbol search
  • Precise refactoring support (rename, call hierarchy)

3 Sub-Agent / Task System

  • Spawn child agent loops for parallel investigation
  • Task delegation with isolated context windows
  • Result aggregation and conflict resolution
  • Resource budgets per sub-agent (tool calls, tokens)

4 Hook System + Path Restrictions

  • Pre/post tool execution hooks (user-defined shell commands)
  • Path-based access restrictions (allowlist/blocklist)
  • Command allowlist/blocklist for shell execution
  • Auto-lint and auto-test hooks on file changes

5 Notebook + Enhanced UX

  • Jupyter .ipynb cell-level read/edit/insert/delete
  • Shift+Tab keyboard shortcut for quick actions
  • Session history search across all sessions
  • Enhanced repo map with dependency graph
  • Auto-detection heuristics (framework, language, project type)

New Ideas for V3+

Beyond Parity

Ideas that would push Opta CLI beyond feature parity into unique competitive territory, leveraging the local-first architecture.

Smart Tool Selection

Lightweight local classifier that predicts which tools the model will need. Pre-loads relevant file context before the model even asks. Reduces round-trips by 30-40%.

Performance

SKILL-CANON Integration

Port Claude Code's 88-skill cannon system. Skills as structured prompts that inject domain expertise (TDD, debugging, brainstorming) into the agent's system prompt on demand.

Capability

Inference Routing

Route tasks to different models based on complexity. Simple edits → small model (fast). Complex architecture → large model (accurate). Semantic router with cost/latency optimization.

Architecture

Screen Capture Tool

Capture screenshots for visual debugging. Feed UI screenshots to vision-capable models. Unique to local-first CLI — no cloud CLIs can access the user's screen.

Unique

Auto-Lint / Auto-Test

Automatically run linter and relevant tests after every file edit. Feed failures back into the agent loop without user intervention. Catches regressions instantly.

Quality

Multi-Model Consensus

Run the same prompt through 2-3 local models, compare outputs, pick the best. Unique to local-first — no API cost per model. Higher accuracy on critical edits.
