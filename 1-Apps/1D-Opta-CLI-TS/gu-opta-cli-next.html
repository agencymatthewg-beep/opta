🛡 Security Hardening

S1: Path Traversal Guard

All file tools (read_file, write_file, edit_file, delete_file) accept arbitrary paths. The model can read /etc/passwd or write to ~/.ssh/authorized_keys. Add assertWithinCwd() that resolves symlinks and validates the path is within the project directory.

Security Critical Impact ~2hr effort

S2: Environment Variable Allowlist

Hooks manager and custom tool loader spread process.env into spawned shell commands, leaking API keys and secrets. Replace with explicit allowlist: PATH, HOME, USER, SHELL, TERM, LANG, OPTA_*. Affects src/hooks/manager.ts:152 and src/tools/custom.ts:95.

Security Critical Impact ~1hr effort

S3: Graceful SIGINT Handler

Current SIGINT handler calls process.exit(130) immediately, leaving MCP servers, background processes, and LSP servers as orphaned zombies. Implement a cleanup registry that gracefully shuts down all subsystems before exit.

Security Refactor High Impact ~4hr effort

S4: Config Validation on Set

opta config set writes values directly without Zod validation. Users can write invalid config (wrong types, impossible values) that crashes subsequent runs. Run the full OptaConfigSchema.parse() before persisting.

Security Medium Impact ~1hr effort
🧪 Testing & Quality

76 test files exist, but many are shallow (schema validation, mock-heavy). Key gaps are integration-level tests for the agent loop and TUI streaming pipeline.

T1: Agent Loop Integration Tests

The core agent loop (src/core/agent.ts, 745 lines) is the heart of Opta. Test: multi-turn conversation flow, tool call → execute → feed-back loop, context compaction trigger, circuit breaker pause, streaming token accumulation. Mock the OpenAI client, not the agent itself.

Testing High Impact ~8hr effort

T2: TUI Adapter + Streaming Tests

The TuiEmitter in src/tui/adapter.ts bridges the agent loop to React state. Test: event emission sequence (connection → waiting → first-token → streaming → done), error propagation, progress timer accuracy, concurrent message handling.

Testing High Impact ~4hr effort

T3: Permission Flow E2E

Test the full permission chain: config-defined permission → runtime resolution → CI mode override → user prompt → tool execution gate. Cover edge cases: permission downgrade mid-session, unknown tool names, mode transitions (safe → auto → dangerous).

Testing Medium Impact ~3hr effort

T4: Slash Command Registry Tests

5 slash handler modules (src/commands/slash/*.ts) have zero direct tests. Test: command parsing, argument extraction, help text generation, model switching flow, session management. These are high user-touch-point features.

Testing Medium Impact ~3hr effort

T5: New TUI Component Tests

Recently added TUI components lack tests: WelcomeScreen, StreamingIndicator (new braille version), adapter.ts events. The existing MessageList.test.tsx was updated but other new components need ink-testing-library coverage.

Testing Low Impact ~3hr effort
✨ New Features

F1: Anthropic Provider Backend

Currently LMX-only. Add Anthropic Claude as a second provider via the existing ProviderClient abstraction. This enables cloud fallback when LMX is down and access to Claude's superior reasoning. The src/providers/anthropic.ts stub already exists.

Feature High Impact ~8hr effort

F2: Streaming Reconnection

When LMX drops mid-stream (as seen with the UN-state freeze), the TUI hangs forever at "Waiting for first token...". Implement: connection health heartbeat (5s interval), automatic retry with exponential backoff, partial response recovery, user notification with manual retry option.

Feature UX High Impact ~6hr effort

F3: Interactive Undo with Checkpoint Picker

Git checkpoints exist (src/git/checkpoints.ts) but there's no interactive UI to browse and restore them. Add /undo slash command that shows a fuzzy-searchable list of checkpoints with diff preview, letting users selectively revert tool changes.

Feature UX Medium Impact ~8hr effort

F4: Code Review Mode (/review)

Aider has /review — Opta doesn't. Add a /review slash command that: reads git diff, sends to LMX with a code review system prompt, renders inline comments with file:line references, suggests fixes as tool calls. Leverages existing git_diff tool and markdown renderer.

Feature Medium Impact ~5hr effort

F5: Session Search & Fuzzy Resume

Session resume requires exact ID. Add fuzzy search over session titles and content, so opta chat --resume "auth fix" finds the right session. Use existing session store + a lightweight Fuse.js index built at session list time.

Feature UX Low Impact ~2hr effort

F6: Architect Mode (/architect)

Planning-only mode that reads the codebase but never writes. The model can use read_file, search_files, list_dir but all write tools are denied. Outputs a plan as markdown. Useful for understanding before modifying.

Feature Low Impact ~2hr effort
🔧 Code Quality & Refactoring

R1: OpenAI Client Singleton

src/core/agent.ts:316-319 creates a new OpenAI client on every agentLoop() call. This wastes memory and prevents connection reuse. Create the client once at session start and pass as a dependency. Also enables connection pooling for sub-agents.

Refactor Performance Medium Impact ~1hr effort

R2: Canonical Token Estimation

Token estimation is duplicated 8 times across agent.ts, subagent.ts, registry.ts, thinking.ts, chat.ts etc. Extract to src/utils/tokens.ts (already exists but not used everywhere). One estimateTokens() function, one formatTokens() function.

Refactor Medium Impact ~2hr effort

R3: Single Permission Source of Truth

Permission defaults exist in both src/core/config.ts (Zod schema) and src/core/tools/permissions.ts (DEFAULT_TOOL_PERMISSIONS). These are already drifting. Delete DEFAULT_TOOL_PERMISSIONS and derive runtime defaults from the Zod schema's .default().

Refactor Medium Impact ~1hr effort

R4: Type-Safe JSON Boundaries

Multiple JSON.parse() as T casts without runtime validation: session loading (memory/store.ts:65), API responses (lmx/client.ts:144), custom tool definitions. Add Zod .safeParse() at all JSON boundaries to catch corrupt data early.

Refactor Security Medium Impact ~3hr effort

R5: Dead Code Removal

~120 lines of dead code identified: die() in errors.ts (never called), InputEditor class with 13 unused methods, fireError imported but never invoked, agent-profiles.ts imported but not functionally wired. Clean removal, no behavior change.

Refactor Low Impact ~1hr effort

R6: Sub-Agent Timeout Cancellation

When a sub-agent times out, the LLM request continues running on the GPU, and filesystem mutations from in-flight tool calls can complete after the timeout. Pass an AbortSignal to the OpenAI client and tool executors so timeout actually cancels work.

Refactor Security Medium Impact ~3hr effort
