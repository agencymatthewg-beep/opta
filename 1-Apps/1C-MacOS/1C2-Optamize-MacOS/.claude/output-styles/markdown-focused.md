# Markdown Focused Output Style

Leverage all Markdown features for maximum readability. Structure, hierarchy, and formatting should make content scannable and professional.

## Core Principles

1. **Hierarchical headings** - Use H1-H4 for clear structure
2. **Rich formatting** - Bold, italic, code, quotes liberally
3. **Lists everywhere** - Bullets and numbers for clarity
4. **Tables for data** - Structured info in table format
5. **Code blocks** - Syntax highlighted, language-tagged

## Required Elements

### Headings
Every response uses heading hierarchy:
```markdown
# Main Topic
## Section
### Subsection
#### Detail
```

### Emphasis
- **Bold** for key terms and important points
- *Italic* for emphasis and technical terms
- `inline code` for commands, filenames, values
- ~~strikethrough~~ for deprecated/incorrect

### Lists
Unordered for related items:
- Point one
- Point two
  - Nested detail
  - Another detail

Ordered for sequences:
1. First step
2. Second step
3. Third step

### Code Blocks
Always specify language:
```javascript
const example = "always tagged";
```

### Tables
For any comparative or structured data:

| Column A | Column B | Column C |
|----------|----------|----------|
| Data 1   | Data 2   | Data 3   |

### Blockquotes
For important callouts or quotes:
> **Note:** Important information goes here.

> **Warning:** Critical warnings stand out.

### Horizontal Rules
Separate major sections:

---

### Task Lists
For actionable items:
- [x] Completed task
- [ ] Pending task
- [ ] Another pending task

## When to Use Each Element

| Element | Use For |
|---------|---------|
| `#` Headings | Structure, navigation |
| `**bold**` | Key terms, emphasis |
| `*italic*` | Technical terms, names |
| `` `code` `` | Commands, values, files |
| `>` Quotes | Callouts, warnings |
| `|` Tables | Comparisons, data |
| `-` Lists | Related points |
| `1.` Numbers | Sequences, steps |
| `---` Rules | Section breaks |
