🚀 Purpose

Opta Init is the front door to the Opta Local stack. The Opta Local ecosystem currently has no public onboarding surface — discovery happens through repos and word-of-mouth. Init creates the definitive first-touch experience: a polished, premium static site that gets any user downloaded, set up, and productive with their local AI stack in minutes.

Not a dashboard. Not a docs site. Exactly one job: get users started.

🔒 Locked Decisions

D01
Static export over SSR
Content never changes at runtime. Zero server cost. Perfect CDN caching. No runtime failure modes.
Locked
D02
Next.js over plain HTML
Ecosystem consistency with 1L, 1F, 1B. TypeScript, next/font, Framer Motion integration. Extensible for v2.
Locked
D03
Framer Motion spring physics only
Spring physics = higher perceived quality. CSS ease-out detectable as "cheap." Interruptible by default.
Locked
D04
Single page with anchor nav
Onboarding page UX, not a docs site. Scroll-through is the correct experience. No navigation overhead.
Locked
D05
Dark mode only at v1
Opta is a dark brand. Quality light mode is significant effort with diminishing returns at v1.
Locked v1
D06
Domain: init.optalocal.com
Matthew's directive. "Init" as in initialize your stack — more meaningful than "get" or "download."
Locked
