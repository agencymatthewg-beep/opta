# Plan 11-02: Design System Compliance Audit

## Overview

Fix remaining design system violations discovered during codebase audit.

## Tasks

### 1. Fix Games.tsx Launcher Colors
**File**: `src/pages/Games.tsx` (lines 57-76)

**Problem**: Uses hardcoded Tailwind colors instead of CSS variables.

**Current (violates design system)**:
```tsx
const LAUNCHER_DISPLAY = {
  steam: {
    color: 'text-blue-400',
    bgColor: 'bg-blue-500/15',
    borderColor: 'border-blue-500/30',
  },
  epic: {
    color: 'text-slate-300',
    bgColor: 'bg-slate-400/15',
    borderColor: 'border-slate-400/30',
  },
};
```

**Fix**: Map to semantic CSS variables:
```tsx
const LAUNCHER_DISPLAY = {
  steam: {
    color: 'text-primary',
    bgColor: 'bg-primary/15',
    borderColor: 'border-primary/30',
  },
  epic: {
    color: 'text-muted-foreground',
    bgColor: 'bg-muted/15',
    borderColor: 'border-border',
  },
  gog: {
    color: 'text-accent',
    bgColor: 'bg-accent/15',
    borderColor: 'border-accent/30',
  },
};
```

### 2. Fix LaunchConfirmationModal Header
**File**: `src/components/LaunchConfirmationModal.tsx` (line 163)

**Problem**: Uses solid `bg-primary/5` instead of glass effect.

**Current**:
```tsx
<div className="px-6 py-4 bg-primary/5 border-b border-primary/10">
```

**Fix**:
```tsx
<div className="px-6 py-4 glass-subtle border-b border-border/30">
```
