Opta LMX Masterclass

The complete, holistic guide to the Local Model eXecution engine.

Opta LMX is the high-performance local inference engine at the heart of the stack. While you might first install the platform using Opta Init, LMX is the actual background daemon that runs on your machine permanently.

It acts as the invisible bridge. If you are typing commands into the Opta CLI, or viewing your active models via the Opta Local Dashboard, they are all simply sending API requests to the LMX background service.

Architecture & Memory

LMX solves the hardest problem in local AI: Out-Of-Memory (OOM) crashes. It does this via a three-tier memory abstraction.

Tier 1: VRAM

GPU Execution

The fastest tier. LMX attempts to load 100% of model layers into your dedicated GPU or Apple Silicon unified memory first.

Tier 2: Sys RAM

CPU Offloading

If VRAM is full, LMX dynamically spills remaining layers to system RAM. Inference slows, but the app never crashes.

Tier 3: Disk

KV Cache Swap

During massive context operations (100k+ tokens), LMX can swap older context states to NVMe storage.

Feature Deep-Dive

Hot-Swapping Models

Unlike traditional setups, LMX stays running even when no models are loaded. You can switch from `llama-3` to `phi-3` via the CLI in milliseconds. LMX gracefully dumps the VRAM and loads the new GGUF file from disk without restarting the port listener.

Concurrent Queuing

If multiple requests hit LMX simultaneously, it batches them safely. This is critical when you use LMX as a backend for complex multi-agent workflows.

Air-Gapped Telemetry

LMX generates extensive performance logs (tokens/sec, load times), but this data is stored strictly locally. If you wish to back up your config across machines, you must actively link it using Opta Accounts.
