---
title: Frontend Design Workflow
description: Iterative workflow for creating and reviewing frontend UI changes
---

1. Receive layout or styling instructions from user.
2. Formulate 3 distinct conceptual approaches.
3. Automatically render these approaches locally as HTML prototypes (e.g. `design-prototypes/logo1.html`, `.../logo2.html`).
4. Ensure components respect standard branding artifacts (`icons` or SVG data).
5. Let user review prototypes and choose path forward before full integration.
