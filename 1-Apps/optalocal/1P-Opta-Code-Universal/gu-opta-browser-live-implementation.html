01 — Full Stack Architecture

CLI Agent
opta chat / opta do — Agent loop dispatches browser_* tool calls via BrowserMcpInterceptor. All calls routed through mcp-interceptor.ts before reaching Playwright.
Policy evaluation + risk scoring + artifact recording
Daemon
BrowserLiveHost (live-host.ts) — HTTP server on ports 46000–47000. Up to 5 session slots. Each slot exposes /frame returning JPEG via Peekaboo screen capture. 500ms frame cache. Auth via viewerAuthToken.
JPEG frames over HTTP polling · 800ms interval · token auth
Browser
@playwright/mcp integration with BrowserMcpInterceptor. Policy engine evaluates risk level. High-risk ops (browser_evaluate, browser_file_upload) are gated. Artifacts (screenshots, snapshots) auto-recorded via artifacts.ts.
Peekaboo capture → macOS screen grab regardless of window focus
Visual
Peekaboo (peekaboo.ts) — macOS screen capture using capturePeekabooScreenPng. Captures Chromium window buffer even when it's in background. Also supports peekabooClickLabel, peekabooTypeText, peekabooPressKey for AI-vision-based interaction. Used by visual-diff.ts pipeline.
Already-working component: LiveBrowserView.tsx polls /frame
Desktop
LiveBrowserView.tsx (already in codebase!) — Polls http://127.0.0.1:{slot.port}/frame every 800ms with token auth. Renders as <img className="browser-frame-img">. Has play/pause/stop controls, URL bar, exponential backoff. browserLiveHostClient.ts scans ports 46000–46009 with 24 concurrent workers.

03 — Browser Visual States (BrowserVisualState)

Idle
No active browser session. Host not running or no session slots occupied.
Active
Browser session running, agent connected, no pending operations.
Working
Agent is executing browser tool calls — streaming events in progress.
Blocked
Pending permission gate — agent waiting for user approval before proceeding.
Derived by deriveBrowserVisualState() in browserVisualState.ts — used by LiveBrowserView to render status indicator.

07 — Implementation Plan

01
Add isLiveViewOpen + isLiveViewExpanded state to App.tsx
Two booleans: isLiveViewOpen (Ctrl+L toggles) and isLiveViewExpanded (Ctrl+Space toggles within live view). When expanded, same CSS class as other full-studio overlays.
src/App.tsx
02
Wire Ctrl+L and Ctrl+Space keybindings
Add to the keyboard handler in App.tsx (same pattern as Ctrl+B/M/A). Ctrl+L toggles isLiveViewOpen. Ctrl+Space, when live view is open, toggles isLiveViewExpanded. Guard: Ctrl+Space only fires when isLiveViewOpen === true.
src/App.tsx — keyboard handler
03
Render LiveBrowserView in chat column when open (compact mode)
In the chat column JSX, add a conditional section above (or replacing) the message list: when isLiveViewOpen && !isLiveViewExpanded, render the existing LazyLiveBrowserView component. The chat list scrolls below. Import already exists as a lazy component.
src/App.tsx — chat column JSX
04
Add full-screen overlay mode (expanded) using existing studio CSS
When isLiveViewExpanded is true, render LiveBrowserView in an overlay with the same opta-studio-overlay class used by BrowserStudio/ModelsStudio/AtpoStudio. Reuses the entrance animation, backdrop, and close button patterns already built for those studios.
src/App.tsx — studio overlay section
05
Add "Start Browser" button to BrowserStudio (Ctrl+B)
BrowserStudio should have a primary CTA button that calls the daemon operation to start a browser session. Once started, the live host will allocate a slot. The user can then press Ctrl+L to see the live feed. Wire via daemonClient.runOperation("browser.start").
src/components/BrowserStudio.tsx (or equivalent)
06
Add Ctrl+L to Command Palette (Ctrl+K)
Register a palette command: { id: "live-browser", title: "Open Live Browser View", run: () => setIsLiveViewOpen(v => !v) }. Makes it discoverable without memorizing the keybinding.
src/App.tsx — palette commands useMemo

08 — Implementation Sketch

// App.tsx additions

// 1. State
const [isLiveViewOpen, setIsLiveViewOpen] = useState(false);
const [isLiveViewExpanded, setIsLiveViewExpanded] = useState(false);

// 2. Keybindings (in existing keyboard handler)
if (e.ctrlKey && e.key === "l") {
  e.preventDefault();
  setIsLiveViewOpen(v => !v);
  if (!isLiveViewOpen) setIsLiveViewExpanded(false);  // reset on re-open
}
if (e.ctrlKey && e.key === " " && isLiveViewOpen) {
  e.preventDefault();
  setIsLiveViewExpanded(v => !v);
}

// 3. Compact mode render (inside chat column JSX)
{isLiveViewOpen && !isLiveViewExpanded && (
  <div className="live-browser-compact">
    <Suspense fallback={<div className="opta-loading">Loading...</div>}>
      <LazyLiveBrowserView
        connection={connection}
        compact={true}
        onExpand={() => setIsLiveViewExpanded(true)}
      />
    </Suspense>
  </div>
)}

// 4. Full studio overlay (same pattern as BrowserStudio/ModelsStudio)
{isLiveViewOpen && isLiveViewExpanded && (
  <div className="opta-studio-overlay opta-studio-live">
    <div className="opta-studio-header">
      <span>Live Browser</span>
      <span className="opta-studio-hint">Ctrl+Space to collapse</span>
    </div>
    <Suspense fallback={null}>
      <LazyLiveBrowserView
        connection={connection}
        compact={false}
        onCollapse={() => setIsLiveViewExpanded(false)}
      />
    </Suspense>
  </div>
)}

// 5. Palette command
{ id: "live-browser", title: "Toggle Live Browser View", run: () => {
    setIsLiveViewOpen(v => !v);
  }
}
