# Opta Home — Marketing & Capabilities

**Tagline:** Opta is the Optimiser. Opta Local is the first public way to run Opta AI.

## 🎯 The Purpose
Opta Home (`optalocal.com`) must do two jobs at once:
- Explain what **Opta** is as a business and product vision.
- Position **Opta Local** as the first public release and practical entrypoint into that vision.

It is not only a local-stack homepage. It is the canonical "What is Opta?" page for new users.

## 🧠 Core Narrative (must be explicit)
- **What Opta is:** Opta builds and sells completely and holistically optimized solutions using AI.
- **What Opta AI is:** Opta AI is the heart and personality of Opta; it is the optimizer users interact with.
- **What Opta Local is:** Opta Local is the first public Opta release, focused on local-first control and performance.
- **How users activate Opta AI for coding:** Users run Opta AI through **Opta CLI** or **Opta Code** after selecting a model runtime.
- **How runtime selection works:** Opta AI can be powered by either an **Opta LMX local model** or a **cloud model**.

## ✨ Core Marketing Points (What makes it impressive?)
- **Company + Product Clarity:** Clearly separates "Opta" (the optimisation company/platform) from "Opta Local" (the first shipping release).
- **Activation Architecture Clarity:** Shows the real flow: `Model Source (LMX or Cloud) → Opta AI → CLI/Code → Optimized outcomes`.
- **Technical Credibility First:** Uses real benchmark data (`22.2 tok/s`, `512GB Unified Memory`) and architecture diagrams over generic AI aesthetics.
- **Ecosystem Orientation:** Separates core user products from management websites so users immediately understand where to go next.
- **Premium Visual Identity:** Keeps the Opta void-native design language (`#09090b`, Electric Violet, Sora + JetBrains Mono, precision motion).

## 🛠 Features & Capabilities
- **Opta Explainer Layer:** Homepage copy must define Opta, Opta AI, Opta Local, and how they relate.
- **Activation Path Visualization:** Dedicated section showing how users power/activate Opta AI in CLI/Code via local LMX or cloud models.
- **Intelligent Routing:** Sends install/setup users to `init.optalocal.com` and active users to operational surfaces (`lmx`, `accounts`, etc.).
- **Architecture Visualization:** Shows local-active architecture and where cloud models can be used without losing Opta workflow control.
- **Terminal DNA Previews:** Demonstrates real CLI/Code usage patterns as proof of practical capability.
- **Mobile-Responsive Precision:** Maintains hierarchy and activation clarity on mobile, not just on desktop.

## ✅ Website Implementation Requirements
- **Hero copy change:** Lead with Opta-level positioning, not only local infrastructure messaging.
- **"What is Opta?" section:** Add a concise company/platform explainer directly under hero.
- **"What is Opta AI?" section:** Explain it as the optimizer personality/interface users work with.
- **"How activation works" section:** Include a clear 4-step flow: choose model runtime, activate Opta AI, use CLI/Code, ship optimized outputs.
- **Ecosystem card updates:** Update product cards so CLI and Code are framed as Opta AI execution surfaces.
- **CTA updates:** Ensure primary CTA points to the activation path, not just generic local infra.
