TUI Architecture ‚Äî Opta CLI vs OpenCode

OPTA CLI Architecture
  • Ink 5 + React 18 ‚Äî React component tree rendering to terminal
  • TypeScript / Node.js runtime (ESM-only)
  • Commander.js for CLI routing + Ink for full-screen TUI
  • --tui flag opts into alternate buffer mode
  • Dual-mode: inline REPL (default) + full-screen TUI
  • 10 TSX components, 2 custom hooks
  • 718 lines of TUI code across 14 files
OPENCODE Architecture
  • SolidJS + custom @opentui renderer ‚Äî reactive UI with Zig/Yoga layout
  • Bun runtime (not Node.js)
  • Client-server architecture (TUI ‚Üî HTTP/SSE ‚Üî headless server)
  • Always full-screen ‚Äî no inline REPL mode
  • 60 FPS render target with Kitty keyboard protocol
  • 16 nested SolidJS context providers
  • ~15,000+ lines of TUI code

TUI Verdict

Opta CLI now has all the structural components of a full-screen TUI ‚Äî split panes, sidebar, scrollable messages, persistent status bar, focus management, and themes. However, OpenCode's TUI is significantly more mature with its custom Zig renderer, dialog system, command palette, Shiki syntax highlighting, momentum scrolling, and 62-property theme engine. The architectural gap is medium ‚Äî Opta has the skeleton, OpenCode has the polish.

Recommended Areas for Improvement

Prioritized by impact on competitive positioning. Critical items block market viability. High items represent significant competitive gaps. Medium items are differentiators. Polish items are nice-to-have refinements.

 Critical ‚Äî Blocks Market Viability
C1 ‚Äî MCP Integration
Implement Full MCP Client
MCP is becoming the standard protocol for AI tool integration. Without it, Opta CLI cannot connect to GitHub, databases, Slack, or any of the 100+ MCP servers in the ecosystem. Both Claude Code and Kimi Code have full MCP. This is the single biggest capability gap.
C2 ‚Äî Web Search & Fetch
Add Web Research Tools
An agent without web access is severely limited for research, documentation lookup, and staying current. Both competitors have built-in web tools. Add web_search and web_fetch tools to the core tool set, or prioritize MCP (C1) which enables this via servers.
 High ‚Äî Significant Competitive Gaps
H1 ‚Äî Dialog/Modal System
Build TUI-Native Dialogs
Currently, pickers (model, session, theme) drop out of the TUI into @inquirer/prompts, breaking immersion. Build an Ink-based dialog component with fuzzy search, backdrop overlay, and keyboard navigation. This is what makes OpenCode feel like a real application.
H2 ‚Äî Git Undo/Redo
Implement Checkpoint & Rewind
Auto-commit agent file changes, track checkpoints, and allow /rewind to restore both conversation and code state. This is Claude Code's most-loved safety feature and essential for trust in autonomous agents.
H3 ‚Äî Hooks System
Add Lifecycle Event Hooks
Pre/post tool execution hooks enable security gates, logging, custom validation, and workflow automation. Start with 4 core events: PreToolUse, PostToolUse, SessionStart, Stop. This unlocks enterprise-grade customization.
H4 ‚Äî Shiki Syntax Highlighting
Replace marked-terminal with Shiki
OpenCode uses Shiki (VS Code's highlighter) for code blocks. The visual difference is dramatic ‚Äî proper language-aware syntax coloring vs. basic ANSI. This single change would make Opta CLI's output look professional-grade. Add line numbers and copy-to-clipboard.
 Medium ‚Äî Differentiators
M1 ‚Äî Command Palette
Add Ctrl+P Fuzzy Command Search
An Ink overlay component that fuzzy-searches all available commands, recently used items, and file references. This is the single most impactful UX improvement for discoverability.
M2 ‚Äî Skills System
Implement SKILL.md Loading
Load markdown skill files from .opta/skills/ and expose them as slash commands. Support frontmatter for tool restrictions and system prompt injection. This enables the community to extend Opta CLI without code changes.
M3 ‚Äî Image Input
Complete Multimodal Support
Wire the existing /image command to actually encode and send images to the model. Support clipboard paste (Ctrl+V) and file path arguments. Many models Opta routes to (GPT-4o, Claude, Gemini) already support vision.
M4 ‚Äî Anthropic Provider
Add Direct Anthropic API Support
The AnthropicClient stub exists. Implementing it would let Opta CLI use Opus 4.6 directly ‚Äî the strongest coding model available (80.9% SWE-Bench). Combined with model agnosticism, this makes Opta the only CLI that can use any model including the best ones.
 Polish ‚Äî Refinements
P1
Toast Notifications
Auto-dismiss overlays for async events (tool complete, error, info).
P2
Momentum Scrolling
macOS-style acceleration in ScrollView for natural feel.
P3
OSC 52 Clipboard
Escape-sequence clipboard integration for remote SSH sessions.
P4
Theme Depth (62 props)
Expand from 6-color themes to OpenCode's 62-property system with dark/light variants.
P5
Prompt Stash
Push/pop prompt drafts for context switching without losing work.
P6

Ctrl+X leader key with 2-second window for chained shortcuts, like OpenCode.
Strategic Position

After the 68-task UX overhaul, Opta CLI has transformed from a basic readline REPL into a legitimate full-screen TUI application with split panes, sidebar, themes, scrollable messages, and rich input handling. Its competitive position is strongest in model flexibility (the only agent that truly works with any LLM) and local-first privacy (zero cloud dependency via Opta-LMX). The next phase should focus on MCP integration and TUI polish (dialogs, Shiki, command palette) to close the remaining gap with OpenCode's visual maturity while maintaining the model-agnostic advantage that neither Claude Code nor Kimi Code can match.
