01 · Foundation
Getting Started Optimally

The essential commands and setup that unlock 80% of Opta's value in the first session.

1

Build & Link Globally

npm run build && npm link — Makes opta available system-wide. Hot reload: npm run dev.

2

Check Your Connection

opta status — Verifies LMX server is reachable. Shows loaded models, memory, and inference speed.

3

Initialize Project Intelligence

opta init — Generates APP.md, AGENTS.md, TASKS.md. The model reads these for every session, giving it deep project context. Do this once per repo.

4

Enable Shell Completions

Add to your ~/.zshrc: eval "$(opta completions zsh)" — Tab completion for commands, flags, and models.

5

Run Health Check

opta doctor — Diagnoses connection, config, tool access, and environment. Catches issues before they bite.

★ Power move: Run opta chat --tui for the full-screen experience. Split panes, real-time token stats, model picker (Ctrl+M), and the new Local Model Insights showing live inference observability.
02 · Modes
Choose the Right Mode

Each mode constrains the model's behaviour for the task at hand. Picking the right mode is the single biggest productivity lever.

ModeTools AvailableBest ForSafety
--plan Read, list, search only Architecture planning, design reviews, understanding codebases ● Maximum
--review Read, search, structured output Code review, security audit, style checking ● Maximum
--research Read, search, web fetch Exploring unfamiliar code, researching APIs, learning ● Maximum
(default) All tools, ask before edits General development — the sweet spot for most work ● Balanced
--auto All tools, auto-approve edits Trusted tasks: refactoring, formatting, bulk changes ● Reduced
--dangerous All tools, no prompts CI/CD pipelines, automated testing, one-shot tasks ● Minimal
★ Optimal pattern: Start in --plan mode to understand the problem, then switch to default mode for implementation, then --review to verify. You can switch modes in-session via /plan, /review, /research.

04 · Workflows
Essential Workflows

The patterns that experienced users rely on for maximum productivity.

Exploring a Codebase

opta chat --research

# In session:
"How does auth work in this project?"
"Show me all API endpoints"
"What patterns does this codebase use?"

# Save findings:
/export research-notes.md

One-Shot Task

opta do "fix the null check in auth.ts"

# Or with flags:
opta do --auto "format all .ts files"
opta do --model qwen2.5-coder "add tests"

# Undo if needed:
opta chat → /undo

Plan-Then-Execute

opta chat --plan
"Design a caching layer for the API"
# Model reads code, writes plan, can't edit

# Switch to implement:
/plan off
"Now implement what we planned"
# Model edits files with your approval

/commit # Auto-commit changes

Code Review Session

opta chat --review
"Review src/core/agent.ts for bugs"
"Check auth flow for security issues"
"Are there any race conditions here?"

# Structured review output with findings,
# severity levels, and suggested fixes

07 · Configuration
Recommended Config Tuning

Key settings to tune for your workflow. All set via opta config set <key> <value>.

model.default
qwen2.5-coder-32b
Set your preferred default model. Avoids picking each session.
permissions.edit_file
allow
Skip edit confirmations in trusted repos. Keep ask for unfamiliar code.
permissions.run_command
ask (default)
Always confirm shell commands. Change to allow only for trusted scripts.
connection.retry.maxRetries
3 (default)
Auto-reconnect attempts when LMX connection drops mid-stream.
safety.circuitBreaker.pauseAt
40 (default)
Pause agent after N tool calls per turn. Prevents runaway loops.
tui.default
true
Always launch in TUI mode. Set false to default to REPL.
insights.enabled
true (default)
Real-time inference insights. Disable for cleaner output.
provider.active
lmx | anthropic
Switch between local LMX and Anthropic Claude API backends.

10 · Quick Reference
Cheat Sheet

Copy-paste commands for the most common operations.

# ── Getting Started ──────────────────────────
opta status                           # check LMX connection
opta doctor                           # full environment health check
opta init                             # generate project intelligence docs

# ── Sessions ─────────────────────────────────
opta chat                             # REPL mode
opta chat --tui                       # full-screen TUI
opta chat --plan                      # read-only architecture mode
opta chat --review                    # structured code review
opta chat --research                  # exploration without edits
opta do "task description"              # one-shot execution

# ── Models ───────────────────────────────────
opta models                           # list loaded models
opta models use qwen2.5-coder        # switch model
opta models info qwen2.5-coder       # model details + context limit

# ── Config ───────────────────────────────────
opta config list                      # show all settings
opta config set model.default qwen2.5 # set default model
opta config set tui.default true     # always launch TUI
opta config set provider.active anthropic # switch to cloud

# ── In-Session Slash Commands ────────────────
/                                     # interactive command browser
/model                                # switch model mid-session
/plan                                 # toggle plan mode
/commit                               # auto-commit staged changes
/undo                                 # interactive checkpoint rollback
/export notes.md                     # export session as markdown
/status                               # show session info
/help                                 # all commands with descriptions
