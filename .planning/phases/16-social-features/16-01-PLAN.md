# Plan 16-01: Score Sharing

wave: 1
autonomous: true

## Overview

Implement minimal, export-only social features that let users share their optimization results OUT of the app. No accounts, no backend complexity - just simple export and share functionality.

**Design Philosophy**: Opta should be convenient, intuitive, powerful, autonomous, efficient, and simplified. No overcrowding, no unnecessary features, no confusion.

## Tasks

### Task 1: ShareCard Component
**File**: `src/components/ShareCard.tsx` (new)

Create a visually appealing score card component:
- Opta branding (logo, colors)
- Overall score prominently displayed
- Hardware tier badge
- 3 dimension scores (Performance, Experience, Competitive)
- Wow factors (money saved, percentile)
- Fixed dimensions optimized for social (1200x630 for Twitter)
- Static rendering (no animations) for html2canvas capture

### Task 2: Image Export Utilities
**File**: `src/lib/shareUtils.ts` (new)

Implement client-side image generation:
- `generateImage()` - Uses html2canvas to convert element to image
- `saveAsImage()` - Download as PNG file
- `copyToClipboard()` - Copy image blob to clipboard
- `shareToTwitter()` - Open Twitter/X web intent
- `copyShareText()` - Copy formatted text for Discord/other

### Task 3: ShareModal Component
**File**: `src/components/ShareModal.tsx` (new)

Modal for sharing options:
- Share to Twitter/X button
- Copy text for Discord button
- Copy image to clipboard button
- Download as PNG button
- Hidden ShareCard rendered off-screen for capture

### Task 4: Connect to Score Page
**File**: `src/pages/Score.tsx`

Wire up the existing TODO handlers:
- `handleShare()` - Open ShareModal
- `handleExport()` - Generate and download score card image
- Enable Share and Export buttons in OptaScoreCard
