Active v0.1.1 Next.js 16 SWR Polling Port 3003 Vercel
What It Does

The LMX Dashboard is the primary web management surface for the Opta LMX inference engine. It reads all data live from the LMX REST API at runtime — no state of its own. Relationship: exactly like Opta Code is to the CLI daemon. It does NOT run inference, does NOT store data, and does NOT require authentication on LAN (trust-LAN model).

Security Model
🔒
timingSafeEqual with length-paddingToken comparison pads both buffers to max(len(a),len(b)) before comparing — prevents timing attacks AND token-length oracle attacks
🔒
Dual auth pathsBearer header for HTTP; ?token=T query param for WebSocket (EventSource can't set headers)
🔒
Loopback-only CORSOrigin regex: localhost / 127.0.0.1 / [::1] only — no external origins accepted
🔒
File permission hardeningstate.json + token + daemon.pid = 0o600; daemon dir = 0o700
🔒
SESSION_ID_RE path traversal guardSession IDs validated by regex before being used in filesystem paths
🔒
Disk headroom checkensureDiskHeadroom() before any daemon dir creation — prevents runaway storage exhaustion
